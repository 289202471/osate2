Reasons for errors when instantiating models

There are the main reasons for having a model that does not instantiate:

  • Connection between two components that inherit features: not supported for now. This error is triggered when you try to connect two components that inherits the same component (and such, share the same inherited features).

Why cannot we use our own AADL_Project file ?

Using your own AADL_Project file would conflict with the files provided by the plug-in. There are possibilities for supporting that but it would rather impact user experience. On the other hand, it could be done by following this process:

  1. Delete the AADL_Project file from the plug-in resources
  2. Create a new project with your own AADL_Project file (let's call it myproperties
  3. Introduce a dependency between Plugin_Resources and myproperties. To do so, select Plugin_Resources, click on the Project menu and then Properties. Finally, in the project references, select myproperties.
